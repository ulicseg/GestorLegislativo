# Documentation Rules

## DO
- Keep docs technical, current, and concise.
- Document deployment-sensitive behavior and project conventions.
- Prefer examples that match the actual repository structure.

## DO NOT
- Do not add decorative fluff.
- Do not document features that are not implemented.

## Quality Gates
A documentation change is valid only if:
- It matches the current codebase.
- It is understandable without extra context.
