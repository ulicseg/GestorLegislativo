# Commits Rules

## DO
- Use atomic commits with a single purpose.
- Keep commit messages imperative and descriptive.
- Stage only the files that belong to the task.

## DO NOT
- Do not bundle unrelated fixes into one commit.
- Do not use vague messages like "update" or "fix".

## Quality Gates
A commit is valid only if:
- The diff is focused.
- The message describes the actual change.
